# LampaTrakt — контекст проекта

## Проект

Репозиторий: `KZuev/LampaTrakt` (публичный).
Основной файл: `trakttv.js` — плагин Trakt.tv для медиа-интерфейса Lampa (3.0+, Maker/ContentRows API).
Общение с пользователем — **на русском**.

Плагин добавляет на главную/категории строки Trakt (Up Next, Watchlist, Recommendations, Calendar), полноэкранные каталоги, scrobbling (отметка просмотренного).

## Текущая версия

**v3.2.64** — Фикс: «Смотреть дальше» и «Хочу посмотреть» на главной иногда пропадали ПОЛНОСТЬЮ при холодном старте (не показывали неверный контент, а просто не появлялись). Причина — тот же механизм кэша строк (`createRowCall`, `trakttv.js:16327`), что и в v3.2.63, но другой триггер: когда кэша строки реально нет (первая установка, сброшенное хранилище, либо кэш протух по TTL 6ч), используется таймаут `UI_DEADLINE_MAIN_MS`/`UI_DEADLINE_CATEGORY_MS`, и если живой фетч (`Api.watchlist()`/`Api.upnext()`) не успевает уложиться — секция для этой загрузки страницы не рисуется вообще (`finish(null)`), а уже отрисованную страницу более поздний успешный ответ не обновляет. `TraktUpNextRow`/`TraktWatchlistRow`/`TraktRecommendationsRow`/`TraktCalendarRow` зарегистрированы с `index: 0` на главной и все стартуют одновременно при холодном старте, гоняясь за одним сетевым всплеском (плюс возможен реальный round-trip обновления токена перед первым запросом, если он истекает). Старые значения (`2800`/`3200` мс) не были ничем обоснованы и, похоже, тюнились только под частый случай (кэш есть → 0 мс задержка). Фикс — увеличены до `6000`/`7000` мс: единственное место использования этих констант — `createRowCall`, и `Lampa.ContentRows.call` обрабатывает каждую строку асинхронно независимо, так что более долгий таймаут не блокирует остальную страницу — просто даёт медленной сети/обновлению токена больше шансов успеть, вместо того чтобы полностью терять секцию до следующего захода на главную.

**v3.2.63** — Фикс: строка «Хочу посмотреть» на главной иногда после холодного старта показывала почти только фильмы из «Скоро» (ожидаемые цифровые релизы), как будто основной список не загрузился. Причина — НЕ гонка в загрузке `trakt_released` (он проставляется синхронно в том же ответе `/sync/watchlist?extended=full`), а отравление кэша строк: `createRowCall` (`trakttv.js:16327`) при наличии кэша всегда показывает его немедленно (`STALE_PRESENT_DEADLINE_MS = 0`), а живой фетч, даже когда он приходит, только обновляет кэш для СЛЕДУЮЩЕЙ загрузки — уже отрисованное не трогает. При этом `saveRowToCache` не проверял качество состава результата (в отличие от `_localeIncomplete`-фикса v3.2.34) — если один-единственный живой фетч watchlist из-за временного сетевого/авторизационного сбоя возвращал заметно урезанный список, фильтр `TraktWatchlistRow` (безусловно пропускающий «ожидаемые» через `getUpcomingMovieIds()`, но отбрасывающий обычные элементы без `trakt_released`) оставлял почти только «ожидаемые» — и этот единичный плохой снимок кэшировался и воспроизводился на каждом холодном старте до 6 часов (`STALE_CACHE_TTL_MS`).

Фикс — общий (не специфичный для watchlist) defensive-guard в `createRowCall`: перед `saveRowToCache` сравниваем число элементов нового результата с числом элементов уже загруженного `staleLine`; если просадка резкая (было ≥4 элементов, стало <40% от прежнего) — не перезаписываем кэш, оставляем прежний хороший до его истечения по TTL, чтобы следующий успешный фетч сам себя исправил. Тот же guard применён и к соседней ветке «пустой отфильтрованный результат» — раньше она безусловно чистила кэш (`clearRowCache`) даже при разовом сбое; теперь чистит только если прежний кэш был маленьким (<4 элементов), иначе считает это временным сбоем и оставляет старый кэш как есть.

**v3.2.62** — Фикс «дозагрузка календаря кидает на самый верх страницы» на iPhone (Apple TV работал нормально). Причина: `loadMoreDays()` (`trakttv.js:7593+`) после добавления новых дней безусловно вызывал `Lampa.Controller.collectionSet(scroll.render())` — это API remote-навигации Lampa, пересобирающее «коллекцию» фокусируемых элементов и по умолчанию фокусирующее первый (самый верхний) элемент коллекции; на touch-платформах `.focus()` на элементе триггерит нативный `scrollIntoView` браузера — отсюда прыжок в начало страницы при каждой дозагрузке. На Apple TV это необходимо (без пересборки коллекции пульт не видит новые элементы), но на iPhone страница листается пальцем, `Controller`-навигация не задействована. Фикс — `Lampa.Controller.collectionSet`/`collectionFocus` теперь вызываются только когда `!Lampa.Platform.screen('mobile')` (уже устоявшийся в файле способ определять touch-платформу, см. `trakttv.js:10295`). На Apple TV поведение не изменилось.

**v3.2.61** — Критический фикс: краш `this.emit is not a function` при открытии Календаря, ломающий навигацию ВСЕГО приложения (не только календарь), сообщалось как «слетела авторизация, ничего не открывается». Причина — гонка в `TraktTimetableAll.create()` (`trakttv.js:7607+`, async-generator): пока `create()` ждёт Trakt+TMDB запросы (может занять секунды), пользователь может уйти с экрана — Lampa вызывает `destroy()`, но `create()` об этом не знает и после резолва промисов всё равно безусловно дёргает `this.activity.toggle()`/`.loader(false)` на уже неактуальной активности. Проверка `this.activity && typeof this.activity.toggle === 'function'` не спасала — метод физически есть, но вызов уходит в Lampa-ядро (`Controller.toggle → Activity.start → start`), которое эту активность больше не считает текущей, и там на внутреннем объекте нет `.emit`. `Controller`/`Activity` — общие синглтоны на всё приложение, поэтому необработанное исключение внутри НИХ (не в коде плагина — трасса шла через `app.min.js`) вешает общую навигацию, а не только календарь.

Фикс: флаг `this._destroyed` (выставляется в `destroy()`), проверка в начале `case 2` — если уже уничтожен, сразу `return` без построения DOM и без обращения к `this.activity`. Плюс defense-in-depth — `this.activity.loader(false)`/`.toggle()` обёрнуты в `try/catch`, чтобы даже непредвиденный похожий сценарий не вешал навигацию, а тихо гасился.

Про авторизацию: невалидный токен НЕ создаёт «плохой объект» напрямую, но `requestApi` при наличии `trakt_refresh_token` делает preflight-проверку/рефреш — то есть протухающая авторизация делает `create()` ДОЛЬШЕ, а значит гонку — заметнее. Ощущение «слетела авторизация» скорее всего было следствием, а не причиной; сама гонка не зависела от состояния авторизации и требовала фикса независимо от него.

**v3.2.55** — Фикс «затемнение после кольца загрузки не пропадает» (гард на повторный вход — не помогло, см. v3.2.56). ## Архитектура scrobbling

**Пути отметки:**
- **Path A (основной)**: `Lampa.Timeline.listener 'update'` → `processTimelineUpdate` → при `progress >= threshold && _isPlayerActive` → `finish(media)` → `addToHistory`
- **Path B (резервный)**: `Lampa.Player.listener 'destroy'` → `setTimeout(500ms)` → `routeFinishIntent` (только если Path A не сработал)
- **Внешний плеер (v3.0.39)**: `Lampa.Player.listener 'external'` (infuse/tvOS Pro/MX и т.д. — встроенный плеер не запускается, `'start'`/`'destroy'` НЕ шлются) → `onPlayerExternal` → `onPlayerStart(data)` (флаг + hash-meta) → при возврате в Lampa приходит финальный Timeline `'update'` → Path A отмечает

**`_isPlayerActive` guard** — НЕ ТРОГАТЬ:
- Защищает от ложных отметок когда `Lampa.Timeline` шлёт `'update'` при открытии списка файлов торрента (с сохранённым прогрессом ≥ порога)
- Снимается в `routeFinishIntent` (при `'destroy'`) для встроенного плеера
- 500 мс задержка в Path B нужна для Apple TV: `destroy` → `_isPlayerActive=false` → Timeline `'update'` 100% приходит ПОСЛЕ; задержка позволяет Timeline сработать пока флаг ещё true

**Внешний плеер — сброс флага (v3.0.39):** т.к. `'destroy'` не приходит, `_isPlayerActive` снимается двухуровнево: (1) основной — сразу после `finish()` в `processTimelineUpdate` при `_externalPlayerActive`; (2) резервный — таймер `EXTERNAL_ACTIVE_WINDOW_MS` (6ч) на случай прерывания ниже порога. `_externalActiveUntil` защищает от сброса при повторном запуске. `'external'` шлётся только при реальном `Player.play`, не при просмотре списка файлов торрента → защита от ложных отметок сохранена.

**`_watchMarkLog`**: массив до 50 записей, хранится в `Lampa.Storage('trakt_watch_log')`, восстанавливается при `watching.init` до первого вызова `_watchLogAdd`. Просматривается через Настройки → Trakt.TV → Отладка → История отметок.

## Авто-торрент — архитектура

1. Кнопка добавляется в `.buttons--container` только при `trakt_at_enabled: true`.
2. **Сериал**: следующий эпизод через `/search/tmdb/{id}?type=show` → `/shows/{traktId}/progress/watched` → `next_episode`.
3. **Сброс фильтра** перед открытием `torrents`: `_atSaveAndClearTorrentFilter()` сохраняет и очищает `torrents_filter` + `torrents_filter_data`, иначе ручной фильтр качества Lampa скрыл бы часть торрентов (например 4K) ещё до `render`-событий. `_atRestoreTorrentFilter()` возвращает фильтр после выбора торрента (идемпотентно).
4. **Сортировка кандидатов** (`Lampa.Listener.follow('torrent', ...)`): `render`-события собираются 400 мс, затем `_atSortTorrentCandidates` строит ВЕСЬ отсортированный список (сезон / озвучка / качество / популярность) для перебора.
5. **Перебор + автовыбор файла** (`Lampa.Listener.follow('torrent_file', ...)`): `_atTryCurrentTorrent` открывает кандидата с таймаутом `AT_ATTEMPT_MS` (30с). Успех — нужный `season`/`episode` (сериал) или лучший файл (фильм) → `_atSucceed()` + `hover:enter`. Неудача (пустой `list_open`, нет нужной серии после `AT_SETTLE_MS`=2.5с, или 30с-таймаут) → `_atTorrentFailed()` → `Lampa.Controller.back()` закрывает модал → ветка `list_close` с флагом `_atRetrying` открывает следующего кандидата. Лимит — `_atMaxAttempts()` (`trakt_at_max_attempts`: all/5/3).

**Прерываемость:** слой-контроллер `trakt_at` (`_atControllerEnable/Disable`, в Lampa нет `Controller.remove` — возврат через `toggle('content')`) с `back: _atCancel` активен между попытками; во время модала Назад отменяет через `list_close` (не `_atRetrying`). `Lampa.Listener.follow('activity', destroy/torrents)` ловит выход с экрана торрентов до выбора. Для мыши/касания в оверлее есть кликабельная кнопка `.trakt-at-overlay__cancel` (`pointer-events:auto` при `pointer-events:none` у оверлея) → `_atCancel`; `_atCancel` закрывает открытый модал (`.modal`) явно, т.к. клик его не закрывает. `_atCancel/_atFailAll` чистят таймеры/состояние, восстанавливают фильтр, прячут оверлей.

**Ключевые паттерны:**
- `$(btn).on('hover:enter', ...)` — обязательно jQuery (меню «Смотреть» активирует через jQuery trigger)
- `_atNormalizeCard(card)` — нормализует вложенные контейнеры
- `_atSearchString(card)` — строит поисковую строку по `parse_lang`
- `_atTitleMatchesSeason(title, season)` — паттерны из фильтра Lampa
- `_atIsDubbed(title)` — паттерн озвучки (voice p==1)
- `_atHasRusAudio(e)` — `languages: ru` или ключевые слова
- `_atSaveAndClearTorrentFilter()` / `_atRestoreTorrentFilter()` — бэкап/сброс/возврат фильтра торрентов; восстановление на всех путях тимдауна (выбор сделан, `list_close`, таймаут, ошибка API, отмена)
- Состояние перебора: `_atTorrentCandidates/_atTorrentIndex/_atAttemptTimer/_atSettleTimer/_atRetrying`; `_atResetState()` — полный сброс

## Важные детали

**Lampa source:**
- `/tmp/lampa-source` — может отсутствовать в новой сессии; клонировать при надобности
- `cf.lampa.mx` недоступен из облачной сессии (403 `host_not_allowed`)

**Механика Lampa (не трогать):**
- `Line.create()` удаляет `.items-line__head` если `!data.title && !has(Icon)` → `prefetchMainRows` должен передавать `displayTitle`
- `more.onVisible` делает `find('.items-line__head').append` без проверки null при `total_pages > 1`
- `Lampa.Player.listener` НЕ отправляет `'stop'`/`'ended'` — только `'start'`, `'destroy'`, `'create'`, `'ready'`, `'external'`
- Timeline `'update'` срабатывает: (1) каждые 2 мин через `saveTimeLoop`, (2) при закрытии плеера через `destroy()` → `saveTimeView()`
- `torrent_file` event `list_open` отправляется ДО `Arrays.extend` файлов — `season`/`episode` ещё не проставлены; слушать `render`
- `Element.prototype.on` в Lampa = `addEventListener`; для jQuery-событий (`hover:enter`) нужно `$(el).on(...)`

**Хуки Agentfy (`.claude/settings.json`):**
- Хуки: `SessionStart`, `PermissionRequest`, `Stop`, `Notification` → POST на `https://webhook.getagentfy.com`
- Токен **не в репо** (репо публичный!) — берётся из env `AGENTFY_TOKEN`, без него `exit 0`
- Проверка в новой сессии: `echo ${AGENTFY_TOKEN:+задан}`

**Git:**
- PR не создавать без явной просьбы
- Пушить в `origin/main` и синхронизировать сессионную ветку:
  `git push origin HEAD:main && git push origin HEAD:SESSION_BRANCH --force`
  где `SESSION_BRANCH` = `git branch --show-current`
